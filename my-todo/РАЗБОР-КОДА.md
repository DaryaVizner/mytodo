# Полный разбор проекта «My ToDo»

Документ для подготовки к защите. Объясняет **каждый язык** и **каждую важную строку**
кода простыми словами. В конце — шпаргалка «как что менять» для ситуационных задач.

---

## Часть 0. Три языка веба за две минуты

Представь, что сайт — это дом.

- **HTML** — это **каркас и стены**: где комната, где дверь, где окно. Отвечает за
  *структуру* и *содержимое*. Это **язык разметки**, а не программирования — он ничего
  не «вычисляет», он просто расставляет элементы по местам.
- **CSS** — это **отделка**: цвет стен, размер окон, расстановка мебели. Отвечает за
  *внешний вид*. Тоже не язык программирования — это язык *стилей* (описаний «как выглядит»).
- **JavaScript (JS)** — это **электрика и механика**: свет включается по кнопке, дверь
  открывается. Отвечает за *поведение* — реакцию на действия пользователя. Это
  **настоящий язык программирования**: в нём есть переменные, условия, циклы, функции.

Правило: **HTML — что это, CSS — как выглядит, JS — что делает.**

## Часть 1. HTML — разбор `index.html`

### Базовые понятия
- **Тег** — команда в угловых скобках: `<p>` (открывающий) и `</p>` (закрывающий).
- **Элемент** — тег вместе с содержимым: `<p>Привет</p>`.
- **Атрибут** — доп. настройка внутри открывающего тега: `<input type="text">`
  (`type` — имя атрибута, `"text"` — значение).
- **Вложенность** — теги кладутся друг в друга, как матрёшки.

### Разбор по строкам

```html
<!DOCTYPE html>
```
Объявление: «это документ HTML5». Всегда идёт первой строкой, включает современный режим браузера.

```html
<html lang="ru">
```
Корневой тег, в него вложено всё. `lang="ru"` — язык страницы русский (важно для
экранных читалок и правильного переноса слов).

```html
<head>
```
«Служебная» часть — её не видно на странице. Здесь настройки, подключение стилей и скриптов.

```html
<meta charset="UTF-8">
```
Кодировка символов. UTF-8 позволяет корректно показывать русские буквы и эмодзи. Без неё — «кракозябры».

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```
**Ключ к адаптивности.** Говорит телефону: «ширину страницы бери равной ширине экрана
устройства, масштаб 1:1». Без этой строки телефон показал бы «десктопную» версию мелким шрифтом.

```html
<meta name="description" content="...">
```
Краткое описание сайта для поисковиков (SEO). На вид страницы не влияет.

```html
<meta http-equiv="Content-Security-Policy" content="default-src 'self'; script-src 'self'; ...">
```
**Главная строка про безопасность (CSP).** Задаёт «политику»: откуда браузеру разрешено
брать ресурсы. `'self'` = только со своего сайта. Разбор директив:
- `default-src 'self'` — по умолчанию всё только со своего домена;
- `script-src 'self'` — скрипты только свои → **чужой/инлайн-скрипт не выполнится** (защита от XSS);
- `style-src 'self'` — стили только свои;
- `img-src 'self' data:` — картинки свои + встроенные через `data:`;
- `object-src 'none'` — запрет `<object>`/Flash и подобного;
- `base-uri 'self'` — запрет подмены базового адреса;
- `form-action 'none'` — форме запрещено отправлять данные куда-либо (у нас отправку
  перехватывает JS, так безопаснее).

```html
<title>My ToDo</title>
```
Текст на вкладке браузера.

```html
<link rel="icon" href="data:image/svg+xml,<svg ...>✅</svg>">
```
Иконка вкладки (favicon). Здесь она нарисована прямо в адресе через `data:`-URI (эмодзи ✅),
поэтому не нужен отдельный файл и лишний запрос к серверу.

```html
<link rel="preload" href="style.min.css" as="style">
<link rel="stylesheet" href="style.min.css">
```
Подключение стилей. **`preload`** (оптимизация) велит браузеру начать качать CSS как можно
раньше, параллельно. Вторая строка `rel="stylesheet"` собственно применяет минифицированный CSS.

```html
<script src="script.min.js" defer></script>
```
Подключение скрипта. **`defer`** (оптимизация) = «скачивай параллельно, но выполни только
после того, как построится вся страница». Поэтому скрипт не тормозит отрисовку и в нём
уже точно доступны все элементы.

```html
</head>
<body>
```
Конец служебной части. **`<body>`** — видимая часть страницы, всё, что видит пользователь.

```html
<main class="app">
```
Главный смысловой блок. **`class`** — «ярлык», по которому к элементу цепляются стили (CSS)
и скрипты (JS). У одного элемента классов может быть несколько.

```html
<header class="app__head">
  <h1 class="app__title">My&nbsp;ToDo</h1>
  <p class="app__sub">Простой список задач · данные хранятся в браузере</p>
</header>
```
Шапка. `<h1>` — главный заголовок (на странице один). `&nbsp;` — «неразрывный пробел»
(не даёт словам «My» и «ToDo» разорваться на разные строки). `<p>` — абзац-подпись.

```html
<form id="add-form" class="add" autocomplete="off">
```
Форма добавления. **`id`** — уникальное имя элемента (в отличие от `class`, `id` должен быть
один на всю страницу); за него «хватается» JS. `autocomplete="off"` отключает подсказки браузера.

```html
<input id="task-input" class="add__input" type="text"
       placeholder="Что нужно сделать?" maxlength="200"
       aria-label="Новая задача" required>
```
Поле ввода. `type="text"` — обычный текст. `placeholder` — серая подсказка внутри пустого
поля. **`maxlength="200"`** — не больше 200 символов (заодно чуть-чуть про безопасность).
`aria-label` — название поля для незрячих. `required` — форму нельзя отправить пустой.

```html
<button class="add__btn" type="submit">Добавить</button>
```
Кнопка. `type="submit"` — по нажатию «отправляет» форму (мы этот момент перехватываем в JS).

```html
<div class="toolbar">
  <div class="filters" role="group" aria-label="Фильтр задач">
    <button class="chip is-active" data-filter="all"    type="button">Все</button>
    <button class="chip"           data-filter="active" type="button">Активные</button>
    <button class="chip"           data-filter="done"   type="button">Выполненные</button>
  </div>
```
Панель. `<div>` — универсальный контейнер-«коробка» без особого смысла, только для группировки.
**`data-filter="..."`** — свой пользовательский атрибут (любой атрибут вида `data-*`). В JS
он читается как `btn.dataset.filter` и говорит, какой фильтр включить. `is-active` — класс
активной кнопки (подсвечена). `type="button"` — обычная кнопка (не отправляет форму).

```html
  <label class="sort">
    <span class="sort__label">Сортировка</span>
    <select id="sort" class="sort__select" aria-label="Сортировка задач">
      <option value="created">Сначала новые</option>
      <option value="alpha">По алфавиту</option>
      <option value="status">По статусу</option>
    </select>
  </label>
</div>
```
`<label>` — подпись к элементу управления. `<span>` — маленький строчный контейнер для текста.
`<select>` — выпадающий список; `<option value="...">` — его пункты. `value` — «внутреннее»
значение пункта, которое читает JS (пользователь видит текст, программа — `value`).

```html
<ul id="task-list" class="list" aria-live="polite"></ul>
```
`<ul>` — маркированный список; сюда JS будет вставлять задачи (`<li>`). Сейчас он пустой.
`aria-live="polite"` — озвучивать изменения списка для незрячих.

```html
<div id="empty" class="empty" hidden>
  <img class="empty__img" src="assets/empty.svg" alt=""
       width="120" height="120" loading="lazy" decoding="async">
  <p>Пока нет задач. Добавьте первую 👆</p>
</div>
```
Блок «пусто». **`hidden`** — по умолчанию спрятан (покажется, только если задач нет; этим
управляет JS). `<img>` — картинка; `alt=""` — пустое описание, т.к. картинка декоративная.
**`loading="lazy"`** (оптимизация) — грузить картинку лениво, только когда она понадобится.
`decoding="async"` — декодировать не блокируя страницу. `width/height` заданы, чтобы не «дёргалась» вёрстка.

```html
<footer class="app__foot">
  <span id="counter" class="counter">0 задач осталось</span>
  <button id="clear-done" class="link" type="button">Очистить выполненные</button>
</footer>
```
Подвал. `<span id="counter">` — счётчик (текст меняет JS). Кнопка удаления всех выполненных.

```html
  </main>
</body>
</html>
```
Закрываем всё в обратном порядке (правило матрёшки: что открыли последним — закрываем первым).

> **Про названия классов.** Стиль `app__head`, `task__text`, `chip is-active` — это методология
> **БЭМ**: `блок__элемент` и `блок--модификатор`. Просто удобное соглашение об именах, ни на что
> техническое не влияет.

## Часть 2. CSS — разбор `style.css`

### Как устроено правило CSS
```css
селектор {
  свойство: значение;
}
```
Пример: `.add__btn { color: #fff; }` → «всем элементам с классом `add__btn` сделать цвет текста белым».

### Виды селекторов (пригодится в ситуационных задачах)
- `.класс` — по классу (точка): `.task`
- `#id` — по id (решётка): `#task-list`
- `тег` — по имени тега: `body`, `h1`
- `*` — вообще все элементы
- `.a .b` — `.b` внутри `.a` (потомок)
- `.a.b` — элемент, у которого сразу оба класса (`is-active` + `chip`)
- `.a:hover` — состояние: при наведении мыши
- `[data-filter]` — по наличию атрибута

### Box-model (модель коробки) — фундамент вёрстки
Каждый элемент — это коробка из 4 слоёв (изнутри наружу):
`content` (содержимое) → `padding` (внутренний отступ) → `border` (рамка) → `margin` (внешний отступ).

### Разбор по блокам

```css
:root {
  --bg: #f4f5f7;
  --accent: #4f6ef7;
  --radius: 14px;
  ...
}
```
`:root` — весь документ. Здесь заданы **CSS-переменные** (начинаются с `--`). Это как настройки
темы в одном месте: задал `--accent` один раз — используешь везде через `var(--accent)`.
Поменял цвет тут — поменялся на всём сайте. `#f4f5f7` — цвет в HEX (RR GG BB).

```css
@media (prefers-color-scheme: dark) {
  :root { --bg: #14161c; --text: #eceef3; ... }
}
```
**`@media`** — медиазапрос, «применяй эти правила только при условии...». Здесь условие —
у пользователя включена тёмная тема системы; тогда переопределяем переменные на тёмные.
Сайт сам переключается в тёмный режим.

```css
* { box-sizing: border-box; margin: 0; padding: 0; }
```
`*` — все элементы. Сбрасываем стандартные отступы браузера (**«обнуление стилей»**).
**`box-sizing: border-box`** — очень важное: ширина элемента считается вместе с padding и
border, а не поверх них. Без этого вёрстка «разъезжается».

```css
body {
  font-family: system-ui, ...;
  background: var(--bg);
  color: var(--text);
  min-height: 100vh;
  display: flex;
  justify-content: center;
  padding: 24px 16px 48px;
}
```
- `font-family: system-ui` — использовать системный шрифт устройства (быстро, без загрузки).
- `background`/`color` — фон и цвет текста берём из переменных.
- `min-height: 100vh` — высота минимум на весь экран (`vh` = 1% высоты окна).
- **`display: flex`** — включает флексбокс (гибкую раскладку) для содержимого body.
- `justify-content: center` — центрирует содержимое по горизонтали.
- `padding: 24px 16px 48px` — внутренние отступы (сверху 24, по бокам 16, снизу 48).

```css
.app { width: 100%; max-width: 560px; }
```
Приложение занимает всю доступную ширину, **но не шире 560px** — на большом экране контент
не растягивается на всю ширину, а на телефоне сжимается. Это основа адаптивности.

```css
.add { display: flex; gap: 8px; margin-bottom: 16px; }
```
Форма — флекс-контейнер: поле и кнопка встают **в ряд**. `gap: 8px` — расстояние между ними.

```css
.add__input { flex: 1; min-width: 0; ... }
```
**`flex: 1`** — «растянись и займи всё свободное место» (поле тянется, кнопка остаётся по размеру).
`min-width: 0` — техническая строка, чтобы длинный текст не ломал раскладку.

```css
.add__input:focus { outline: none; border-color: var(--accent); }
```
`:focus` — когда поле в фокусе (кликнули/перешли табом). Убираем стандартную обводку и красим
рамку в акцентный цвет.

```css
.add__btn { background: var(--accent); color: #fff; border: none;
            border-radius: var(--radius); cursor: pointer; ... }
```
Кнопка: фон акцентный, текст белый, без рамки, скруглённые углы (`border-radius`),
курсор-«пальчик» (`cursor: pointer`).

```css
.add__btn:hover { opacity: .9; }
.add__btn:active { transform: translateY(1px); }
```
`:hover` — при наведении слегка прозрачнее. `:active` — в момент нажатия кнопка «вдавливается»
на 1px вниз (`transform: translateY`).

```css
.toolbar { display: flex; flex-wrap: wrap; justify-content: space-between; ... }
```
Панель: флекс. `flex-wrap: wrap` — если не влезает по ширине, элементы переносятся на новую
строку. `space-between` — раскидать элементы по краям, промежуток между ними.

```css
.chip { border-radius: 999px; ... }
.chip.is-active { background: var(--accent-weak); color: var(--accent); font-weight: 600; }
```
`.chip` — кнопка-«таблетка» (`border-radius: 999px` делает её полностью скруглённой).
`.chip.is-active` — селектор «оба класса сразу»: активную кнопку подсвечиваем.

```css
.list { list-style: none; display: flex; flex-direction: column; gap: 8px; }
```
Список без маркеров (`list-style: none`), задачи идут **колонкой** (`flex-direction: column`)
с промежутком 8px.

```css
.task { display: flex; align-items: center; gap: 12px;
        box-shadow: var(--shadow); animation: pop .18s ease; }
```
Одна задача: флекс в ряд, `align-items: center` — выровнять по вертикали по центру
(чекбокс, текст, крестик на одной линии). `box-shadow` — тень. `animation: pop` — проиграть
анимацию `pop` за 0.18 сек.

```css
@keyframes pop {
  from { opacity: 0; transform: translateY(-4px); }
  to   { opacity: 1; transform: translateY(0); }
}
```
**`@keyframes`** — описание анимации: от прозрачного и чуть смещённого вверх (`from`) к
видимому на месте (`to`). Новая задача «выезжает».

```css
.task--done .task__text { color: var(--muted); text-decoration: line-through; }
```
У выполненной задачи (класс `task--done`) текст блёкнет и **зачёркивается** (`line-through`).

```css
.task__del:hover { color: var(--danger); background: var(--accent-weak); }
```
Крестик удаления при наведении краснеет (`--danger`).

```css
@media (max-width: 480px) {
  .toolbar { flex-direction: column; align-items: stretch; }
  ...
}
```
**Главный адаптивный блок.** Условие: ширина экрана ≤ 480px (телефон). Тогда панель
перестраивается в колонку и элементы растягиваются на всю ширину — на телефоне удобнее.

```css
@media (prefers-reduced-motion: reduce) {
  * { animation: none !important; transition: none !important; }
}
```
Если у пользователя в системе включено «уменьшить движение» — отключаем все анимации
(забота о людях, которым анимации мешают). `!important` — «это правило главнее прочих».

## Часть 3. JavaScript — разбор `script.js` построчно

### Базовые понятия JS
- **Переменная** — «коробка» с именем, где лежит значение: `var x = 5;`
- **Тип данных**: строка `'текст'`, число `42`, логический `true/false`, массив `[1,2,3]`,
  объект `{ключ: значение}`, `null` (пусто).
- **Функция** — сохранённый набор команд, который запускают по имени: `function f() {...}`,
  вызов — `f()`. Может принимать **аргументы** и **возвращать** результат через `return`.
- **Массив** — список значений: `['a','b']`. У него есть **методы**: `push`, `filter`, `map`, `sort`, `find`.
- **Объект** — набор пар «ключ-значение»: `{ id: 1, done: false }`. Обращение: `task.done`.
- **DOM** — «дерево» страницы: JS видит HTML как объекты, которые можно менять.
- **Событие** — то, что происходит (клик, ввод, отправка формы). На событие «вешают» функцию-обработчик.

### Разбор

```js
'use strict';
```
Строгий режим. Запрещает опасные «вольности» языка и ловит больше ошибок. Хорошая практика.

```js
(function () {
  ...
})();
```
**IIFE** — «сразу вызываемая функция». Мы обернули весь код в функцию и тут же её запустили
`()`. Зачем: все переменные внутри становятся **локальными** и не «протекают» в глобальную
область, не конфликтуют с другими скриптами. Это про безопасность и чистоту.

```js
var STORAGE_KEY = 'my-todo-tasks';
```
Переменная-константа: имя (ключ), под которым задачи хранятся в localStorage. В одном месте —
удобно менять.

```js
var tasks  = load();
var filter = 'all';
var sort   = 'created';
```
**Состояние приложения** — три переменные, описывающие «что сейчас».
`tasks` — массив задач (загружаем из хранилища функцией `load`). `filter` — активный фильтр.
`sort` — активная сортировка.

```js
var form  = document.getElementById('add-form');
var input = document.getElementById('task-input');
... (list, empty, counter, sortSelect, clearBtn)
var filterBtns = document.querySelectorAll('[data-filter]');
```
Находим элементы страницы и сохраняем ссылки на них, чтобы дальше не искать заново.
- **`document.getElementById('...')`** — найти один элемент по `id`.
- **`document.querySelectorAll('[data-filter]')`** — найти **все** элементы с атрибутом
  `data-filter` (это три кнопки фильтра). Возвращает список.

```js
function load() {
  try {
    var data = JSON.parse(localStorage.getItem(STORAGE_KEY));
    return Array.isArray(data) ? data : [];
  } catch (e) {
    return [];
  }
}
```
Загрузка задач из хранилища.
- `localStorage.getItem(KEY)` — прочитать строку по ключу (localStorage хранит только строки).
- `JSON.parse(...)` — превратить строку обратно в массив/объект.
- `Array.isArray(data) ? data : []` — **тернарный оператор** `условие ? да : нет`: если это
  правда массив — вернуть его, иначе пустой массив `[]`.
- `try/catch` — «попробуй, а если ошибка — не падай»: если в хранилище мусор, возвращаем `[]`
  (пустой список) вместо поломки.

```js
function save() {
  localStorage.setItem(STORAGE_KEY, JSON.stringify(tasks));
}
```
Сохранение. `JSON.stringify(tasks)` — превратить массив в строку. `setItem(KEY, строка)` —
записать в localStorage. **Именно это выполняет требование ТЗ «хранение в localStorage».**

```js

  var clean = text.trim();
  if (!clean) return;
  tasks.push({ id: Date.now(), text: clean, done: false, created: Date.now() });
  save();
  render();
}
```
Добавить задачу.
- `text.trim()` — убрать пробелы по краям.
- `if (!clean) return;` — если после обрезки пусто, `!clean` = «истина», выходим и ничего не
  добавляем (защита от пустых задач).
- `tasks.push({...})` — добавить в конец массива **объект задачи**: `id` (уникальный номер,
  `Date.now()` = текущее время в миллисекундах), `text` (текст), `done: false` (не выполнена),
  `created` (когда создана — для сортировки).
- `save()` — сохранить, `render()` — перерисовать список.

```js
function toggleTask(id) {
  var t = tasks.find(function (t) { return t.id === id; });
  if (t) { t.done = !t.done; save(); render(); }
}
```
Переключить «выполнено».
- `tasks.find(...)` — найти в массиве первую задачу, у которой `id` совпадает.
  `===` — строгое сравнение (и значение, и тип).
- `if (t)` — если нашли. `t.done = !t.done` — **инвертировать**: было `false` → станет `true`
  и наоборот (`!` — логическое «не»).

```js
function deleteTask(id) {
  tasks = tasks.filter(function (t) { return t.id !== id; });
  save();
  render();
}
```
Удалить задачу. **`filter`** создаёт новый массив, оставляя только те элементы, для которых
функция вернула `true`. Здесь условие `t.id !== id` — «оставить все, кроме той, что удаляем»
(`!==` — «не равно»). Результатом заменяем `tasks`.

```js
function clearDone() {
  tasks = tasks.filter(function (t) { return !t.done; });
  save();
  render();
}
```
Очистить выполненные. Оставляем только те, у кого `done` = ложь (`!t.done`), т.е. невыполненные.

```js
function visibleTasks() {
  var arr = tasks.slice();
  if (filter === 'active') arr = arr.filter(function (t) { return !t.done; });
  if (filter === 'done')   arr = arr.filter(function (t) { return t.done; });

  if (sort === 'alpha')   arr.sort(function (a, b) { return a.text.localeCompare(b.text); });
  if (sort === 'status')  arr.sort(function (a, b) { return a.done - b.done; });
  if (sort === 'created') arr.sort(function (a, b) { return b.created - a.created; });
  return arr;
}
```
Готовит список к показу с учётом фильтра и сортировки — **не меняя оригинал**.
- `tasks.slice()` — сделать **копию** массива (чтобы сортировка не портила исходный порядок).
- Блок фильтра: если выбран `active` — оставить невыполненные; если `done` — выполненные;
  если `all` — ничего не отсекаем.
- Блок сортировки. **`sort`** сравнивает пары `a` и `b`; если функция вернула отрицательное
  число — `a` идёт раньше `b`.
  - `a.text.localeCompare(b.text)` — сравнение строк по алфавиту (с учётом языка).
  - `a.done - b.done` — по статусу (`false`=0, `true`=1, значит невыполненные выше).
  - `b.created - a.created` — по дате, **новые сверху** (обрати внимание: `b - a` = по убыванию).
- `return arr` — вернуть готовый список.

**Главная функция — перерисовка списка.**
- `list.textContent = ''` — очистить список. **Важно для безопасности**: не `innerHTML`,

- `items` — задачи с учётом фильтра/сортировки.

**`forEach`** — цикл: выполнить функцию для каждой задачи. Для каждой:

```js
function plural(n, forms) {
  var a = Math.abs(n) % 100, b = a % 10;
  if (a > 10 && a < 20) return forms[2];
  if (b > 1 && b < 5)   return forms[1];
  if (b === 1)          return forms[0];
  return forms[2];
}
```
Русское склонение: «1 задача», «2 задачи», «5 задач». `%` — остаток от деления. Правила:
11–19 → «задач»; оканчивается на 2–4 → «задачи»; на 1 → «задача»; иначе → «задач».

## Часть 4. Шпаргалка для ситуационных задач

**«Добавьте кнопку редактирования задачи»**
В `render()`, рядом с кнопкой удаления, создай ещё одну кнопку и повесь на неё:
```js
edit.addEventListener('click', function () {
  var next = prompt('Новый текст:', task.text);
  if (next && next.trim()) { task.text = next.trim(); save(); render(); }
});
```

**«Смените цвет/тему»** — в `style.css` меняешь переменные в `:root` (например, `--accent`).
Один цвет — меняется всюду.

**«Сделайте счётчик всех задач, а не только невыполненных»** — в `render()`
`var left = tasks.length;`

**«Добавьте новый фильтр»** — добавь `<button data-filter="что-то">` в HTML и обработай новое
значение в `visibleTasks()`.

**«Докажите защиту от XSS»** — добавь задачу `<img src=x onerror="alert(1)">`. Ничего не
всплывёт → покажи в DevTools, что `<` превратился в `&lt;`. Причина — `textContent` вместо `innerHTML`.

**«Где localStorage?»** — функции `save()` / `load()`; в DevTools → Application → Local Storage → ключ `my-todo-tasks`.

**«Что за оптимизация?»** — `defer` на скрипте, `preload` на стиле, минифицированные
`.min.css`/`.min.js`, `loading="lazy"` на картинке, данные в localStorage.

**«Почему безопасно?»** — CSP-мета-тег ограничивает источники и запрещает инлайн-скрипты;
ввод выводится через `textContent`; `innerHTML` не используется; длина ввода ограничена `maxlength`.

### Мини-словарь терминов
- **Тег / элемент / атрибут** — кирпичики HTML.
- **Селектор / свойство / значение** — из чего состоит CSS-правило.
- **Переменная / функция / массив / объект / событие** — из чего состоит логика JS.
- **DOM** — страница глазами JS (дерево объектов).
- **Флексбокс** (`display:flex`) — гибкая раскладка в ряд/колонку.
- **Медиазапрос** (`@media`) — правила по условию (размер экрана, тема).
- **CSP** — политика безопасности: откуда грузить ресурсы.
- **XSS** — атака внедрением чужого кода; защита — `textContent`, экранирование, CSP.
- **localStorage** — хранилище браузера (строки, остаются после перезагрузки).
